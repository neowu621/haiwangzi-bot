# 未來優化清單（暫不做、待觀察）

若實測有需要再回來看；沒需要就放著。

## 🚀 效能優化（v267 之後若仍覺得慢）

### A. Service Worker 預載
- 用 SW 快取常用靜態資源（JS/CSS bundle、圖示）
- iPhone 進站時資源直接從 SW 拿，省網路 RTT
- **工程成本**：高（需要寫 SW、處理 update 邏輯）
- **預期效益**：首次後進站省 200-500ms
- **觸發條件**：若實測 v267 仍有客戶抱怨慢，再評估

### B. SSR（Server-Side Rendering）
- 把 LIFF 部分頁面改用 Next.js SSR 預渲染
- 客戶端 hydrate 時不必再 fetch 資料
- **工程成本**：高（要把 client component 拆 server + client）
- **預期效益**：首屏顯示瞬間
- **副作用**：SSR 後 LIFF SDK 初始化邏輯要改、auth flow 變複雜
- **觸發條件**：客戶數 > 1000，效能成本貴於工程成本時

### C. 骨架卡片加細節
- 現在的 skeleton 是純色塊
- 可改成「假的訂單卡片輪廓」，包含日期框框、地點欄、按鈕位置
- **工程成本**：低（純 UI 改動）
- **預期效益**：等待感「視覺上」變舒服（即使實際秒數沒變）
- **觸發條件**：簡單的 UX 升級，有空就做

### D. localStorage cache 加 timestamp + TTL
- 目前 /liff/my 的 cache 沒有過期機制
- 加 timestamp，超過 24 小時就忽略 cache 強制重抓
- **工程成本**：低
- **觸發條件**：發現有人看到「3 天前的訂單」顯示太久才更新

### E. R2 圖片 lazy load
- /liff/my 訂單卡片裡的付款證明圖片現在 eager load
- 改 IntersectionObserver lazy load，只有捲到才下載
- **工程成本**：低
- **預期效益**：首屏更快，省流量

## 🌊 天氣回報強化

### O. 接 CWA 第二個 dataset（自動氣象站）抓潛點實地測站
- 目前 daily-weather-report 只接 O-A0001-001（一般氣象站，僅 ~100 站）
- 鼻頭角 / 龍洞 / 三貂角 / 卯澳這些**潛點所在地**的自動測站可能在 O-A0003-001（自動氣象站，~1000 站，C 字頭 ID 如 C0R190）
- **工程成本**：中（要 union 兩個 API 回傳，欄位略不同）
- **預期效益**：海況數據從「附近 20-30km」精細到「就在潛點」
- **觸發條件**：對風速 / 海況精度有更高需求時

### P. 接潮汐 / 浪高 / 海溫 API
- 目前只有風速 + 氣溫
- 浪高（CWA F-A0085-002）更直接反映「能不能下水」
- 海溫對裝備建議（要不要厚 wetsuit）有用
- **觸發條件**：客戶開始問「今天浪幾米」時

## 🚧 已延後的中型功能（明確規劃但暫不做）

### v272 (已延後) — 教練排班 + 客戶看到誰帶團
**做了會解決**：客戶不知道帶團教練是誰；教練不知道自己排了哪些班
**內容**：
- Schema 加 `DivingTrip.assignedCoachIds: String[]`
- `/admin/trips` 編輯頁加教練 picker（從 coach 用戶清單勾選）
- LIFF 場次頁顯示「👨‍🏫 教練 A · B」
- LIFF 詳情頁顯示每位教練頭像 + 證照 + 評價（評價依賴 v273）
- 教練 `/liff/coach` 加「我今天的場次」自動篩 assignedCoachIds.includes(me)
- 加「請假」按鈕設 `coach.unavailableDates: Date[]`

**觸發條件**：教練數 > 3 人、客戶開始問「誰帶我」時

### v273 (已延後) — 客戶評價系統
**做了會解決**：沒有客戶反饋資料；無法挑出好教練
**內容**：
- Schema 加 `Review { id, bookingId, overallStars, siteStars, perCoach: Json[], publicVisible, createdAt }`
- 訂單 status=completed 隔天 cron 推 LINE Flex 邀客戶評分
- LIFF 評價頁：整體 / 教練 / 海況 三軸星星 + 文字評語（per coach）
- Admin 後台看：每位教練平均星數、評語列表
- LIFF 首頁公開展示好評 carousel
- 依賴 v272（要先有教練排班才能 per-coach 評價）

**觸發條件**：v272 上線後 1-2 個月有累積後做
