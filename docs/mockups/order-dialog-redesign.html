這版的結構（依你的需求）

① #金額：總金額 / 已付款 / 剩餘款 三欄一目了然（剩餘款紅字）。
② #付款紀錄：每一筆時間 + 金額 + 類型 badge（綠＝實收、藍＝抵用金、黃＝折抵/減免），可逐筆刪。
③ #訂單狀態：單一下拉。
④ #金額調整：選「調整項目」（轉帳/現金/LINE Pay/抵用金/老闆折抵/助教減免/其他）→ 選「其他」才出現說明欄 → 填金額（≤剩餘）→ 確認 → 自動寫入付款紀錄。

👉 其他建議

A. 統一模型最大好處：原本「新增付款」和「助教減免」是兩套，現在合併成「金額調整＝一筆有類型的紀錄」，邏輯一致、畫面少一半。
B. 區分「實收 vs 折抵」：badge 用顏色分（綠=真的收到錢、黃=折抵沒收現金）。這對報表/結帳很重要——老闆要知道實際現金多少、折抵多少。建議報表分開算。
C. 每筆紀錄記「誰操作 + 時間」：折抵/減免是敏感動作，自動寫進 audit。
D. 資料面要新增「付款明細表」：目前系統只有單一 paidAmount 數字＋付款憑證圖，沒有結構化的逐筆紀錄。要做這個「付款紀錄」清單，需新增一張 PaymentEntry 資料表（每筆：金額/類型/說明/操作者/時間）。這是這版唯一較大的工程。
E. 刪除某筆紀錄：應寫一筆反向紀錄（紅字 -金額）而非真刪，保留軌跡。
F. 上限驗證：金額 > 0 且 ≤ 剩餘款；折抵類也一樣（避免折到負數）。
